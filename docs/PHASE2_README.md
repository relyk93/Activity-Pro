# ActivityPro — Phase 2 Plan

> Power features: turning a working app into a professional tool for serious facilities.

---

## Goals

Phase 1 proved the concept. Phase 2 makes ActivityPro indispensable — adding features that facility directors and their staff need every day and that justify the Pro/Enterprise tiers.

---

## Features to Build

### 1. Staff Login & Role-Based Access

**Why:** Right now the entire facility shares one account. Real facilities need accountability — who scheduled what, who took attendance, who wrote a note.

**What to build:**
- Login screen with username + password
- Two roles: **Director** (full access) and **Floor Staff** (limited to calendar, attendance, ratings)
- Session management via Streamlit `st.session_state`
- Staff account management page (Director only)
- Activity log showing which staff member did what

**Files to add/modify:** `utils/auth.py`, new `pages/login.py`, new `pages/staff_management.py`

---

### 2. Photo Documentation

**Why:** Regulators and families want proof that activities happened. A photo attached to a session record is worth more than a checkbox.

**What to build:**
- Camera upload or file picker on the rate_activities page
- Store photos linked to a specific event + date
- Photo gallery view per resident and per activity
- Include thumbnails in PDF resident reports

**Files to add/modify:** `pages/rate_activities.py`, `utils/database.py` (add photos table), `utils/pdf_export.py`

---

### 3. Mood & Engagement Trend Graphs

**Why:** The app already collects mood scores (1–5) per session. Nobody is visualizing that data. A 90-day trend line per resident is a powerful care tool.

**What to build:**
- Line chart: resident mood over time (30 / 60 / 90 day views)
- Bar chart: attendance rate by activity type
- Heatmap: which days/times see highest engagement
- Export charts to PDF with care plan reports

**Files to add/modify:** `pages/reports.py` (add chart section using `st.line_chart` / Plotly)

---

### 4. Clinical-Grade PDF Exports

**Why:** The current print system uses HTML-to-print (browser-dependent, inconsistent). Real care plan documents need reliable, formatted PDFs that can be attached to resident files.

**What to build:**
- Replace HTML-to-print with `reportlab` or `weasyprint` for true PDF generation
- Resident care plan PDF: demographics, disability profile, 90-day engagement summary, mood trend chart
- Monthly activity report PDF: attendance totals, top activities, staff notes summary
- Download button in-app (no browser print dialog)

**Files to add/modify:** `utils/pdf_export.py`, `pages/print_calendar.py`, add `reportlab` to `requirements.txt`

---

### 5. EHR Integration

**Why:** Senior living facilities run on Electronic Health Record platforms — primarily **PointClickCare** and **MatrixCare**. If activity data doesn't flow into the EHR, staff have to enter it twice. Double entry is how errors happen.

**What it means in practice:**
- ActivityPro pushes attendance and engagement records to the resident's chart in the EHR
- Care plan notes generated by AI can be copied directly into the EHR note field
- Resident demographics (name, DOB, room number, diagnoses) can be pulled from the EHR instead of entered manually

**What to build:**
- PointClickCare API client (OAuth 2.0, REST) — read residents, write activity notes
- MatrixCare API client — same concept, different auth/endpoint format
- Settings page toggle: "EHR Provider" dropdown (None / PointClickCare / MatrixCare)
- Sync status indicator per resident
- Error handling for API failures (queue and retry)

**Files to add:** `utils/ehr_pointclickcare.py`, `utils/ehr_matrixcare.py`, modify `pages/settings.py`, modify `utils/database.py` (add EHR resident ID field)

> Note: Both PointClickCare and MatrixCare require a partner/developer agreement to access their APIs in production. For development, use their sandbox environments.

## Database Changes Needed

```sql
-- Staff accounts
CREATE TABLE staff (
    id INTEGER PRIMARY KEY,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    role TEXT NOT NULL,  -- 'director' or 'staff'
    full_name TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Activity photos
CREATE TABLE activity_photos (
    id INTEGER PRIMARY KEY,
    event_id INTEGER REFERENCES scheduled_events(id),
    resident_id INTEGER REFERENCES residents(id),
    photo_path TEXT NOT NULL,
    taken_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    staff_id INTEGER REFERENCES staff(id)
);

-- EHR sync tracking
ALTER TABLE residents ADD COLUMN ehr_id TEXT;
ALTER TABLE residents ADD COLUMN ehr_provider TEXT;
```
